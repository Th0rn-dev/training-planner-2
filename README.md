## Планировщик тренировок

Планировщик изначально задумывался как UI каталог доступа к большому количеству видео файлов с разъяснением 
технических действий - карточки с описанием, превью и ссылкой на видео файл.

Для систематизации карточки разбиты на категории.

Далее возникла необходимость планирования тренировок (в процессе разработки).

Сама тренировка будет содержать:
 - дату
 - набор карточек (что тренируем)
 - длительность тренировки по каждой карточке (сколько времени тренируем каждое действие по карточке)
 - Обратную связь (текстовая информация по каждой тренировке: успели/не успели проработать)

Ожидание - статистика по тренировкам должна помочь оценивать процесс эффективности тренировок, на что делать акцент,
не упускать технические действия, которые не были включены в тренировку (это основной момент). Все участники процесса
смогут видеть запланированное и понимать, что предстоит тренировать (есть моменты, что постоянно приходиться вспоминать 
технические действия, которые ранее прорабатывались)


Планы на перспективу:
 - Переход на API (убрать из приложения прямой доступ к БД, авторизация, внедрение ролей)
 - UI клиент на других ЯП (JS, Kotlin) (не зависеть от платформы, на которой UI будет запускаться)
 - Размещение планировщика на сайте сообщества.
